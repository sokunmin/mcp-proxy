# MCP Proxy Development Guide

## Project Overview

**mcp-proxy** is a Python tool that enables switching between MCP (Model Context Protocol) server transports. It provides bidirectional proxying capabilities between stdio and SSE/StreamableHTTP transports with a revolutionary flexible Docker system for automatic package management.

### Key Information
- **Language**: Python 3.10+ (supports 3.10, 3.11, 3.12, 3.13)
- **Framework**: Built on the `mcp` library with `uvicorn` for HTTP serving
- **License**: MIT
- **Version**: 0.8.0
- **Package Manager**: UV (modern Python package manager)
- **Type Checking**: Fully typed with mypy strict mode
- **Terminal**: PowerShell (default as specified in CLAUDE.md)

### Core Functionality
- **Mode 1**: stdio to SSE/StreamableHTTP - Expose stdio MCP servers as SSE/HTTP endpoints
- **Mode 2**: SSE/StreamableHTTP to stdio - Connect to remote SSE/HTTP MCP servers via stdio interface
- **Named Servers**: Support for multiple named server configurations with JSON-based configuration
- **Docker Support**: Multi-arch container images (linux/amd64, linux/arm64) with revolutionary automatic MCP package installation system
- **Flexible Package Management**: Zero-code-change system for adding new MCP servers - just update servers.json and rebuild

## Adding New MCP Servers - Zero-Code-Change Process

### Revolutionary 3-Step Process
1. **Update Configuration**: Add entry to `servers.json` with appropriate command and args
2. **Rebuild Docker Image**: Run `docker build -t mcp-proxy .` - packages automatically installed
3. **Deploy**: Use new image with pre-installed packages - **NO CODE CHANGES REQUIRED**

### Real-World Example: Adding GitHub Integration
```json
{
  "mcpServers": {
    "fetch": {
      "disabled": false,
      "command": "uvx",
      "args": ["mcp-server-fetch"]
    },
    "github": {
      "enabled": true,
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "transportType": "stdio"
    },
    "filesystem": {
      "enabled": true,
      "command": "uvx",
      "args": ["mcp-server-filesystem", "--base-dir", "/workspace"],
      "transportType": "stdio"
    }
  }
}
```

### What Happens During Build
1. `install_mcp_packages.py` parses the updated `servers.json`
2. Detects new packages: `@modelcontextprotocol/server-github`, `mcp-server-filesystem`
3. Installs via `npm install -g` and `uv tool install` respectively
4. Packages are available immediately when container starts
5. No runtime downloads or configuration needed

## Troubleshooting & Debugging

### Common Issues and Solutions
- **Package installation failures**: Check `install_mcp_packages.py` output during Docker build with `--progress=plain`
- **Runtime package not found**: Verify package was properly installed during build using container inspection
- **Permission issues**: System automatically handles permissions - app user has access to pre-installed packages
- **Configuration parsing errors**: Use `validate_solution.py` to test configuration before building
- **Build cache issues**: Use `--no-cache` flag to force fresh package installation
